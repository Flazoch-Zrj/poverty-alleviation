## 1. 文档概述

### 1.1 项目背景

构建一个面向 **贫困户/帮扶对象、帮扶责任人、社会工作者/志愿者、系统管理员** 四种角色的扶贫帮扶管理平台，实现对贫困家庭档案管理、帮扶结对、走访记录、项目管理、捐赠管理、就业帮扶、技能培训、志愿服务、信息公开等业务的全面信息化支撑。

### 1.2 设计目标

- 满足多角色、多业务的扶贫帮扶全流程管理需求
- 基于 Java 1.8 + Spring Boot + Vue 3 技术栈，确保系统稳定可靠
- 前后端分离架构，便于团队协作与持续迭代
- 完善的权限体系，保障数据安全与合规

## 2. 技术栈选型

### 2.1 后端技术

| 类别 | 技术 | 版本 / 说明 |
|------|------|-------------|
| **核心框架** | Spring Boot | 2.7.x（兼容 Java 1.8），依赖注入与自动配置 |
| **安全与认证** | Spring Security + JWT | 用户认证、接口权限校验、Token 无状态鉴权 |
| **持久层框架** | MyBatis-Plus | 简化 CRUD、分页查询、条件构造器、代码生成器 |

| **代码生成** | MyBatis-Plus Generator | 快速生成 entity、mapper、service、controller 基础代码 |

## 3. 系统架构设计

### 3.1 整体架构（前后端分离）

```
┌─────────────────────────────────────────────────────┐
│                    客户端（浏览器）                     │
│  Vue 3 + Element Plus + ECharts + Axios              │
└──────────────────────┬──────────────────────────────┘
                       │ HTTP / HTTPS
                       │ JSON (RESTful API)
                       ▼
┌─────────────────────────────────────────────────────┐
│                  Nginx 反向代理                        │
│         静态资源托管 / API 路由转发 / SSL 终止           │
└──────────────────────┬──────────────────────────────┘
                       │
                       ▼
┌─────────────────────────────────────────────────────┐
│            Spring Boot 后端服务（Java 1.8）            │
│                                                      │
│  ┌──────────┐ ┌──────────┐ ┌───────────────────┐   │
│  │Controller│→│ Service  │→│ Mapper (MyBatis+) │   │
│  └──────────┘ └──────────┘ └────────┬──────────┘   │
│                                     │               │
│  ┌────────────────────────────┐    │               │
│  │ Spring Security + JWT  Filter │    │               │
│  └────────────────────────────┘    │               │
│                                     │               │
│  ┌────────────────────────────┐    │               │
│  │ Redis (Cache / Rate Limit) │    │               │
│  └────────────────────────────┘    │               │
└──────────────────────┬──────────────────────────────┘
                       │
          ┌────────────┼────────────┐
          ▼            ▼            ▼
       MySQL        Redis       Aliyun OSS
       (主库)       (缓存)     / MinIO (文件)
```

```
│   ├── handler         // 登录成功/失败/无权限处理器

├── modules
│   ├── system          // 系统管理模块
│   │   ├── controller  // 用户/角色/权限管理
│   │   ├── service
│   │   └── mapper
│   ├── poverty         // 贫困户档案模块
│   ├── assistance      // 帮扶结对与措施模块
│   ├── visit           // 走访记录模块
│   ├── project         // 扶贫项目模块
│   ├── need            // 困难需求模块
│   ├── donation        // 捐赠管理模块
│   ├── job             // 就业帮扶模块
│   ├── training        // 技能培训模块
│   ├── volunteer       // 志愿服务模块
│   ├── news            // 信息公开模块
│   └── dashboard       // 数据统计看板模块
└── generator           // 代码生成器配置
```

## 6. 表关系与核心逻辑

### 6.1 核心业务关系图

```
┌─────────────────────────────────────────────────────────────────────┐
│                        用户与权限体系                                │
│  sys_user ──→ sys_user_role ──→ sys_role ──→ sys_role_permission   │
│                                                            │        │
│                                                     sys_permission  │
└─────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────┐
│                     帮扶业务核心链路                                 │
│                                                                      │
│  sys_user (cadre) ──→ assistance_pairing ──→ poverty_family         │
│                           │                          │              │
│                           │                    family_member        │
│                           ├── assistance_measure                     │
│                           └── visit_record                          │
└─────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────┐
│                     需求与捐赠链路                                    │
│  poverty_family ──→ needs_publish ──→ donation_money                │
│                                        └── donation_goods           │
│                                                                      │
│  poverty_family ──→ poverty_project ──→ project_audit_log           │
│                          │                                           │
│                    donation_money (项目资金)                          │
└─────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────┐
│                     就业与培训链路                                    │
│  sys_user ──→ job_position ──→ job_application (贫困户)              │
│                                                                      │
│  sys_user ──→ training_course ──→ training_enrollment (贫困户)       │
└─────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────┐
│                     志愿服务链路                                      │
│  sys_user ──→ volunteer_activity ──→ volunteer_record               │
└─────────────────────────────────────────────────────────────────────┘
```

### 6.2 核心业务逻辑

#### 用户与角色

用户通过 `sys_user_role` 多对多关联角色，角色通过 `sys_role_permission` 多对多关联权限。系统采用 Spring Security + JWT 实现无状态认证，每次请求携带 Token，后端通过过滤器解析并校验权限。

#### 结对与帮扶

`assistance_pairing` 联结帮扶干部与贫困户，一对多关系。一户可对应多条 `assistance_measure`（帮扶措施）和 `visit_record`（走访记录），形成完整的"一户一策"帮扶档案。

#### 需求与捐赠

贫困户通过 `needs_publish` 发布需求，志愿者或管理员通过 `donation_money` / `donation_goods` 登记外部捐赠。捐赠可关联到具体需求（定向）或扶贫项目，支持匿名/实名两种方式。

#### 项目流程

帮扶干部提出 `poverty_project` 项目申请，经管理员通过 `project_audit_log` 审核（通过/驳回），审核通过后进入实施/验收状态。项目资金可通过捐赠记录汇入。

#### 就业与培训

帮扶责任人发布 `job_position`，贫困户通过 `job_application` 在线申请岗位。管理员发布 `training_course`，贫困户通过 `training_enrollment` 报名并签到。

#### 志愿服务

志愿者报名 `volunteer_activity` 并签到签退，系统通过 `volunteer_record` 自动计算服务时长，支持志愿者排行。

### 6.3 角色定义与职责

| 角色 | 角色标识 | 核心职责 |
|------|---------|----------|
| **贫困户/帮扶对象** | `poor` | 查看个人档案与帮扶措施，发布困难需求，申请就业岗位，报名技能培训，提交救助申请 |
| **帮扶责任人**（扶贫干部） | `cadre` | 管理结对贫困户档案，记录走访日志，制定跟踪帮扶措施，发起扶贫项目申请，推荐就业 |
| **社会工作者/志愿者** | `volunteer` | 参与志愿活动，记录服务时长，对接社会资源（登记捐赠、物资），协助发布需求，组织培训活动 |
| **系统管理员** | `admin` | 用户与角色管理，项目审核，数据字典维护，捐赠公示审核，数据统计看板，系统配置，操作日志审计 |

## 7. 接口设计规范

### 7.1 通用规范

- **请求方式**：遵循 RESTful 风格，使用 GET / POST / PUT / DELETE
- **数据格式**：请求与响应统一为 `application/json`
- **响应结构**：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {},
  "timestamp": 1234567890
}
```

- **分页请求**：`?page=1&size=10`
- **分页响应**：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "records": [],
    "total": 100,
    "pages": 10,
    "current": 1,
    "size": 10
  }
}
```

### 7.2 接口命名规范

```
/api/v1/{模块名}/{资源}[/{id}][/子资源]
```

示例：
- `GET    /api/v1/system/user/{userId}` — 获取用户详情
- `POST   /api/v1/poverty/family` — 创建贫困户档案
- `PUT    /api/v1/assistance/measure/{id}` — 更新帮扶措施
- `DELETE /api/v1/job/position/{id}` — 删除岗位

### 7.3 状态码约定

| 状态码 | 说明 |
|--------|------|
| 200 | 操作成功 |
| 400 | 请求参数错误 |
| 401 | 未认证（Token 缺失/过期） |
| 403 | 无权限 |
| 404 | 资源不存在 |
| 429 | 请求过于频繁（触发限流） |
| 500 | 服务器内部错误 |

## 8. 安全设计

### 8.1 认证与授权

- **JWT Token 认证**：用户登录后发放 Token（含用户 ID、角色、过期时间），后续请求通过 Header `Authorization: Bearer <token>` 传递
- **Spring Security 过滤器链**：自定义 `JwtAuthenticationFilter` 解析 Token 并设置安全上下文
- **RBAC 权限模型**：基于角色-权限的细粒度接口访问控制，通过 `@PreAuthorize("hasPermission('sys:user:add')")` 注解控制
- **密码安全**：使用 BCrypt 加密存储，禁止明文密码

### 8.2 数据安全

- **参数校验**：使用 Hibernate Validator 对输入参数进行合法性校验，防止 SQL 注入与 XSS
- **敏感数据脱敏**：身份证号、手机号等敏感字段在日志与前端展示时部分脱敏
- **防 SQL 注入**：MyBatis-Plus 参数化查询，禁止拼接 SQL

### 8.3 接口防护

- **接口限流**：自定义 `@RateLimit` 注解 + Redis 实现，对登录、短信验证码等接口进行频率限制
- **CORS 配置**：限定允许的跨域来源
- **文件上传**：限制文件类型与大小，上传至 OSS/MinIO 而非本地服务器
